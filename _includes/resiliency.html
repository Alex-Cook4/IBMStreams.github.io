Resiliency

How resilient is Streams?

We have customers that have been running for over 2 years without downtime, and that was before we introduced high availability. If systems go down, Streams self-heals while Guaranteed Tuple Processing makes sure that no data is lost. Streams provides enterprise resiliency with simple set-up and configuration.

What happens when a node goes down?

The controller service will automatically restart processing elements and services on the other Streams hosts. If a node configured for High Availability comes back online, services will be automatically restarted.

Will I lose data if a node goes down?

In Streams 4.0, we introduced consistent regions, which not only guarantees that the application does not lose data, but, in most cases, ensures that each incoming record is processed exactly once. Consistent regions are enabled using a simple annotation (@consistent) and HA compliant operators. Read more about consistent regions.

What happens if an Application Processing Element fails?

Failures are detected and Processing Elements (PEs) are automatically restarted by the controller service. If needed, the PE will be relocated to a new host.

How do I configure High Availability?

Most HA features, such as automatic service and PE restarts, are enabled by default. You can read configuration documentation here or watch Step 4 in our "Setting up an Enterprise Streams Environment" video series.
